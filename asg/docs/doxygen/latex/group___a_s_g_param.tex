\hypertarget{group___a_s_g_param}{}\doxysection{Parametric objects}
\label{group___a_s_g_param}\index{Parametric objects@{Parametric objects}}


A\+SG follows the principle that different rendering A\+P\+Is expect different parameters for defining objects such as lights, cameras, materials, etc. As an example, a classic, fixed-\/function oriented rasterization-\/centric A\+N\+A\+RI device might support material and lighting models that align with the traditional Blinn-\/\+Phong model and delta light sources, while a ray tracing engine will support physically based materials and lights. A\+SG acknowledges this and allows the user to specify named parameters of different scalar, vector, or sampler types to allow for highest flexibility. \mbox{\hyperlink{struct_a_s_g_param}{A\+S\+G\+Param}} serves as a base type for such specialized parametric object types.
